# 桌面级具身智能机器人感知-决策-执行一体化原型系统需求文档

## 1. 项目概述

### 1.1 项目目标
设计并实现一个基于 PC 端的模块化具身智能 Agent 原型系统，通过“感知-决策-执行”三层解耦架构，完成用户自然语言指令驱动的机械臂物体分拣闭环任务。

### 1.2 核心设计原则
- **模块化解耦**：感知、决策、执行三层独立开发，通过标准接口（MCP）通信。
- **模型可替换**：决策层 LLM 与感知层 VLM 支持在前端界面动态选择与切换。
- **平台化与可扩展**：基于 MCP 协议设计工具集，后续可灵活添加新技能。

### 1.3 技术选型总览
| 层级 | 技术方案 | 备注 |
|------|----------|------|
| **决策编排** | LangGraph (StateGraph) | 实现多 Agent 节点闭环流转 |
| **通信协议** | MCP (Model Context Protocol) | 标准化工具调用与资源访问 |
| **感知VLM** | 轻量云端模型 (MiniMax MCP视觉 / OpenAI GPT-4o) | 可前端切换 |
| **执行VLA** | SmolVLA (固定，不可前端替换) | 封装为 MCP 技能工具 |
| **机器人框架** | LeRobot | 统一硬件控制与数据格式 |
| **前端界面** | Gradio / Streamlit | 轻量、快速搭建原型 UI |
| **硬件平台** | PC + 单机械臂 + 臂上摄像头 | 完全本地部署（除云端 API 调用） |

### 2.2 数据流闭环
1. 用户输入自然语言指令 → 前端发送至决策层。
2. 决策层 **任务规划Agent** 调用 LLM 生成任务序列。
3. **场景分析Agent** 通过 MCP 调用感知层 `describe_scene` 工具获取当前场景描述。
4. **动作决策Agent** 综合任务与场景，通过 MCP 调用执行层工具（如 `run_smolvla`）。
5. 执行层 MCP Server 加载 SmolVLA 模型，根据当前图像和任务生成动作序列，通过 LeRobot 发送至机械臂。
6. **闭环验证Agent** 再次调用感知层工具，判断任务完成度，决定继续执行或结束。

## 3. 模块详细设计

### 3.1 感知层

#### 3.1.1 职责
- 采集原始传感器数据（RGB 图像、机械臂关节状态）。
- 根据决策层需求，通过 MCP 工具对外提供结构化信息。

#### 3.1.2 MCP 工具定义

| 工具名称 | 功能描述 | 输入参数 | 输出 |
|----------|----------|----------|------|
| `get_image` | 采集当前相机图像 | 无 | 图像 (base64) |
| `get_robot_state` | 获取机械臂当前关节角度、末端位姿 | 无 | `{joint_positions, ee_pose}` |
| `describe_scene` | 调用配置的 VLM 模型生成场景描述 | `image` (base64), `prompt` (可选) | 场景描述文本 |

#### 3.1.3 模型可替换设计
- `describe_scene` 工具内部读取前端配置的 VLM 供应商及 API 密钥，动态创建客户端。
- 支持的 VLM 供应商：`minimax_mcp_vision`、`openai_gpt4o`、`ollama_vision`（本地备用）。
- 配置文件示例：
  ```yaml
  perception:
    vlm_provider: "minimax_mcp_vision"
    vlm_model: "minimax-mcp-vision-latest"
    api_key: "${MINIMAX_API_KEY}"

## 3.2 决策层 (毕设重点)
#### 3.2.1 职责
- 接收用户自然语言指令，编排多 Agent 协作流程。
- 通过 MCP Client 调用感知层和执行层工具。
- 维护任务状态，实现闭环反馈控制。
### 3.2.2 LangGraph 状态定义
  ```python
  class AgentState(TypedDict):
    user_instruction: str          # 原始用户指令
    task_queue: List[str]          # 分解后的任务序列
    current_task: str              # 当前正在执行的任务
    current_image: str             # 当前图像 (base64)
    robot_state: dict              # 机械臂状态
    scene_description: str         # VLM 生成的场景描述
    action_result: str             # 执行结果状态 (success / in_progress / failed)
    iteration_count: int           # 当前闭环迭代次数
    conversation_history: List     # 对话历史
  ```

### 3.2.3 Agent 节点设计

| 节点名称 | 功能 | 调用的 MCP 工具 | 模型依赖 |
|----------|------|------------------|----------|
| `task_planner` | 解析用户指令，生成任务序列 | 无 | 前端选定的 LLM |
| `scene_analyzer` | 获取当前场景描述 | `describe_scene`, `get_robot_state` | 前端选定的 VLM |
| `action_decider` | 根据任务和场景决定动作 | `run_smolvla` 或原子工具 | 规则引擎 + LLM |
| `verifier` | 验证任务完成状态 | `describe_scene`, `get_image` | 前端选定的 VLM |

### 3.2.4 LangGraph 图结构

```python
from langgraph.graph import StateGraph, END

builder = StateGraph(AgentState)
builder.add_node("task_planner", task_planner_node)
builder.add_node("scene_analyzer", scene_analyzer_node)
builder.add_node("action_decider", action_decider_node)
builder.add_node("executor", executor_node)      # 调用 MCP 执行工具
builder.add_node("verifier", verifier_node)

builder.set_entry_point("task_planner")
builder.add_edge("task_planner", "scene_analyzer")
builder.add_edge("scene_analyzer", "action_decider")
builder.add_edge("action_decider", "executor")
builder.add_edge("executor", "verifier")
builder.add_conditional_edges(
    "verifier",
    lambda state: "scene_analyzer" if state["action_result"] == "in_progress" else END
)
```

### 3.2.5 LLM 模型可替换设计

- 前端提供下拉框选择 LLM 供应商（MiniMax、OpenAI、Ollama）。
- 决策层启动时读取配置文件，创建对应的 LLM 客户端实例。
- 用户可输入自定义 API Key 或选择本地模型路径。

## 3.3 执行层

### 3.3.1 职责

- 将决策层的动作指令转化为机械臂运动。
- 封装 SmolVLA 模型作为核心抓取技能，并提供基础原子运动工具。

### 3.3.2 MCP 工具定义

| 工具名称 | 功能描述 | 输入参数 | 内部实现 |
|----------|----------|----------|----------|
| `move_to` | 移动到指定空间位置 | `x`, `y`, `z`, `orientation`(可选) | 逆运动学求解 + 轨迹规划 |
| `move_home` | 返回预设初始位置 | 无 | 读取预设关节角度 |
| `grasp` | 闭合夹爪 | `force`(可选) | 调用夹爪驱动接口 |
| `release` | 打开夹爪 | 无 | 调用夹爪驱动接口 |
| `run_smolvla` | 执行基于 SmolVLA 的抓取技能 | `task_description`, `current_image`, `robot_state` | 加载 SmolVLA 模型，生成并执行动作序列 |

### 3.3.3 SmolVLA 封装细节

- 固定不可替换：SmolVLA 作为系统预设技能，不在前端提供切换选项。
- 推理流程：
  - 接收任务描述文本、当前图像、机械臂状态。
  - 调用 SmolVLA 模型推理，输出末端位姿增量序列（或直接输出关节角度）。
  - 通过 LeRobot 的 `robot.send_action()` 接口逐帧发送动作。
- 扩展性：未来可通过添加新的 MCP 工具（如 `run_placement_vla`）来引入其他技能模型，保持架构开放。

## 3.4 前端界面

### 3.4.1 功能模块

- 任务交互区：
  - 用户自然语言输入框
  - 执行状态实时显示（当前任务、场景描述、动作日志）
  - 摄像头实时视频流展示
- 模型配置面板：
  - 决策 LLM 选择：下拉框（MiniMax / OpenAI / Ollama），支持 API Key 输入。
  - 感知 VLM 选择：下拉框（MiniMax MCP Vision / OpenAI GPT-4o / Ollama Vision），支持 API Key 输入。
  - 系统载入模型：显示当前加载的执行层模型（SmolVLA，不可修改）。
- MCP 工具扩展面板：
  - 展示当前已注册的 MCP 工具列表。
  - 支持用户手动刷新工具列表（为后续动态添加工具预留接口）。
- 个性化设置：
  - 机械臂初始位置校准
  - 执行速度缩放系数
  - 最大闭环迭代次数

### 3.4.2 智能体辅助

- 模型部署助手 Agent：当用户选择“用户部署模型”时，通过对话引导用户完成 API Key 配置或本地模型路径指定。
- 系统载入助手 Agent：在系统启动时自动检测可用模型，并在前端下拉框中填充选项。

## 4. 开发与部署计划

### 4.1 并行开发路线

| 阶段 | 主线任务 (决策系统) | 支线任务 (数据与模型) | 预计耗时 |
|------|----------------------|------------------------|----------|
| 第一阶段 | 搭建 LangGraph + MCP 框架骨架 | 安装 LeRobot，录制 50-100 组分拣动作数据 | 2-3 周 |
| 第二阶段 | 实现各 Agent 节点与状态流转 | 使用录制数据微调 SmolVLA 模型 | 4-5 周 |
| 第三阶段 | 完成前端界面开发与 MCP 工具集成 | 将 SmolVLA 封装为 MCP 工具，测试推理 | 2-3 周 |
| 第四阶段 | 端到端闭环调试，撰写论文 | 系统稳定性优化，准备答辩演示 | 2-3 周 |

### 4.2 数据采集方案

- 任务场景：桌面物体分拣（如将不同颜色方块分类放入指定区域）。
- 采集方式：通过 LeRobot 提供的 `control_robot.py` 脚本进行人工示教。
- 数据量：50-100 个成功轨迹，涵盖不同物体位置和抓取角度。
- 数据格式：LeRobot 标准格式（Parquet + 视频帧）。

### 4.3 测试验证

- 单元测试：各 MCP 工具独立调用测试。
- 集成测试：LangGraph 图在模拟 MCP 工具下的逻辑正确性。
- 实物测试：完成至少 10 次连续成功的分拣任务，记录成功率。

### 6.2 MCP Server 注册示例

```python
# mcp_servers/perception_server.py
from mcp.server import Server
import cv2

server = Server("perception-server")

@server.call_tool()
async def get_image() -> str:
    img = camera.capture()
    return image_to_base64(img)

@server.call_tool()
async def describe_scene(image: str) -> str:
    vlm_client = get_vlm_client()  # 从配置读取
    return await vlm_client.describe(image, prompt="描述场景中的物体位置和状态")
```
