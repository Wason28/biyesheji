# 03 系统架构

## 1. 架构目标

本项目采用“感知-决策-执行”三层解耦架构，确保模块独立开发、标准接口通信和后续技能扩展。

## 2. 分层架构

### 2.1 前端界面

负责：
- 模型选择
- 任务输入
- 状态监控
- MCP 工具扩展面板

### 2.2 决策层

技术基线：
- `LangGraph + MCP Client`

负责：
- 多 Agent 节点编排
- 状态流转
- 条件分支
- 闭环控制

### 2.3 感知层

技术基线：
- `MCP Server + VLM`

负责：
- 图像采集
- 机器人状态获取
- 场景描述生成

### 2.4 执行层

技术基线：
- `MCP Server + SmolVLA + LeRobot`

负责：
- 原子动作执行
- 高层抓取技能执行
- 机械臂动作下发

## 3. 数据流闭环

主流程固定为：
1. 用户输入自然语言指令。
2. 决策层拆解任务。
3. 决策层调用感知层获取场景信息。
4. 决策层调用执行层产生并执行动作。
5. 决策层再次验证任务状态。
6. 根据验证结果继续闭环或结束任务。

## 4. 架构约束

- 层间通信通过 `MCP` 进行。
- 决策层不直接操作硬件。
- 前端不直接承担核心业务决策。
- 感知层和执行层对外只暴露标准工具，不泄露内部实现细节。

## 5. 可扩展点

- 新增感知工具
- 新增执行技能工具
- 新增前端展示组件
- 新增模型供应商适配器

扩展方式要求：
- 优先新增 `MCP` 工具，不破坏既有主流程。
- 新增能力必须同步更新接口文档和测试文档。
