# 04 感知层详细规范

## 1. 职责

感知层负责：
- 采集原始 RGB 图像。
- 采集机械臂关节状态和末端位姿。
- 基于配置的 VLM 生成场景描述。
- 通过 `MCP` 工具向决策层输出结构化信息。

## 2. MCP 工具

### 2.1 `get_image`

功能：
- 采集当前相机图像。

输入：
- 无

输出：
- 当前图像的 `base64` 编码

### 2.2 `get_robot_state`

功能：
- 获取机械臂当前关节角度和末端位姿。

输入：
- 无

输出：
- `{joint_positions, ee_pose}`

### 2.3 `describe_scene`

功能：
- 调用当前配置的 VLM 生成场景描述。

输入：
- `image`
- `prompt`，可选

输出：
- 场景描述文本

## 3. 模型适配要求

- `describe_scene` 必须从配置中读取当前 VLM 供应商和模型。
- 支持的 VLM 供应商：
- `minimax_mcp_vision`
- `openai_gpt4o`
- `ollama_vision`
- 不允许在核心逻辑中写死具体厂商。

## 4. 输出要求

- 感知输出应尽量结构化、稳定、可复用。
- 场景描述应重点覆盖物体类别、位置、相对关系和抓取状态。
- 感知结果必须能直接为决策层节点使用。

## 5. 错误处理

- 摄像头采集失败时必须返回明确错误。
- 机器人状态获取失败时必须中止本轮依赖状态的决策调用。
- VLM 调用失败时必须记录错误信息并可用于闭环回退。

## 6. 测试要求

- 每个工具至少具备成功与失败两个测试用例。
- `describe_scene` 至少验证两种模型配置路径。
- 应具备模拟输入测试，避免硬件不可用时完全无法开发。
