# 04 感知层详细规范

## 1. 职责

感知层负责：
- 采集原始 RGB 图像。
- 采集机械臂关节状态和末端位姿。
- 基于配置的 VLM 生成场景描述。
- 通过 `MCP` 工具向决策层提供稳定合同。

## 2. MCP 工具

### 2.1 `get_image`

功能：
- 采集当前相机图像。

输入：
- 无

对决策层的稳定输出：
- 图像 `base64`

### 2.2 `get_robot_state`

功能：
- 获取机械臂当前关节角度和末端位姿。

输入：
- 无

对决策层的稳定输出：
- `{joint_positions, ee_pose}`

### 2.3 `describe_scene`

功能：
- 调用当前配置的 VLM 生成场景描述。

输入：
- `image`
- `prompt`，可选

对决策层的稳定输出：
- 场景描述文本

说明：
- 底层实现可以附带额外元数据，但不得破坏上述最小可用合同。

## 3. 模型适配要求

- `describe_scene` 必须从配置读取当前 VLM 供应商和模型。
- 支持的 VLM 供应商：
  - `minimax_mcp_vision`
  - `openai_gpt4o`
  - `ollama_vision`
- 不允许在核心逻辑中写死具体厂商。

## 4. 输出要求

- 感知输出应稳定、可复用。
- 场景描述应优先覆盖物体类别、位置、相对关系和抓取状态。
- 感知结果必须能直接服务决策层节点。

## 5. 错误处理

- 摄像头采集失败时必须返回明确错误。
- 机器人状态获取失败时不得继续依赖该状态推进决策。
- VLM 调用失败时必须返回可追溯错误信息，便于闭环回退和排查。

## 6. 测试要求

- 每个工具至少具备成功与失败两个测试用例。
- `describe_scene` 至少验证两种模型配置路径。
- 应具备模拟输入测试，避免硬件不可用时完全无法开发。

## 7. 第二阶段边界

允许变更：
- 更换 `VLM provider`、相机适配器和机器人状态适配器。
- 在不破坏最小合同前提下补充元数据与结构化观察字段。
- 增强层内校验、错误处理和 provider 管理。

禁止变更：
- 输出动作建议、任务拆解结果或闭环控制结论。
- 耦合前端展示逻辑或执行层内部实现。
- 破坏 `get_image`、`get_robot_state`、`describe_scene` 的最小输出合同。
- 绕过统一 `ToolEnvelope` 直接暴露底层 provider 私有返回结构。
