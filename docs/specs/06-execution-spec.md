# 06 执行层详细规范

## 1. 职责

执行层负责：
- 将决策层动作指令转换为机械臂运动。
- 提供基础原子工具。
- 封装 `SmolVLA` 作为核心抓取技能。

## 2. MCP 工具

| 工具名称 | 功能描述 | 输入参数 | 内部实现 |
|----------|----------|----------|----------|
| `move_to` | 移动到指定空间位置 | `x`, `y`, `z`, `orientation` | 逆运动学求解 + 轨迹规划 |
| `move_home` | 返回预设初始位置 | 无 | 读取预设关节角度 |
| `grasp` | 闭合夹爪 | `force` | 调用夹爪驱动接口 |
| `release` | 打开夹爪 | 无 | 调用夹爪驱动接口 |
| `run_smolvla` | 执行基于 `SmolVLA` 的抓取技能 | `task_description`, `current_image`, `robot_state` | 模型推理 + 动作序列执行 |

## 3. `SmolVLA` 约束

- `SmolVLA` 为固定核心执行技能。
- 当前版本不在前端提供切换入口。
- 推理输出可为末端位姿增量序列或关节角度。
- 动作下发必须通过 `LeRobot` 接口执行。

## 4. 执行流程

`run_smolvla` 的基本流程：
1. 接收任务描述、当前图像、机器人状态。
2. 加载或获取当前 `SmolVLA` 模型实例。
3. 生成动作序列。
4. 通过机器人接口逐帧发送动作。
5. 返回执行结果状态。

## 5. 安全要求

- 所有动作工具必须进行参数校验。
- 机械臂运动范围必须配置化管理。
- 通讯失败或动作失败时必须停止后续动作。
- 实物联调阶段必须具备安全停止手段。

## 6. 扩展要求

- 后续扩展技能时，通过新增 `MCP` 工具实现。
- 不通过替换当前 `run_smolvla` 实现新技能扩展。

## 7. 测试要求

- 原子工具必须支持独立测试。
- `run_smolvla` 必须支持模拟输入测试。
- 真实机械臂联调结果应记录到 `docs/records/EXPERIMENTS.md` 或 `docs/records/TEST_REPORT.md`。
