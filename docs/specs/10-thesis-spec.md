# 10 论文撰写规范

## 1. 文档目的

本项目为毕业设计，论文工作必须与开发工作并行推进。本规范用于约束 Thesis Agent 和其他开发 Agent 如何持续沉淀论文材料。

## 2. 章节映射

| 论文章节 | 对应内容 |
|----------|----------|
| 绪论 | 具身智能背景、模块化架构意义、本文贡献 |
| 相关工作 | LeRobot 生态、LangGraph 多 Agent 系统、SmolVLA 轻量化 VLA 模型 |
| 系统架构设计 | 三层解耦架构图、MCP 协议集成方案 |
| 决策层 Agent 设计 | LangGraph 状态图、节点功能、闭环反馈机制、模型可替换实现 |
| 系统实现与验证 | 硬件平台、数据采集流程、实物实验设置、任务成功率分析 |
| 总结与展望 | 平台化设计的扩展性、未来可添加的 MCP 工具与技能 |

## 3. Thesis Agent 职责

- 从第一阶段开始维护论文材料
- 每个里程碑后更新论文章节草稿
- 每次实验后更新实验数据和结果分析
- 维护图表、截图、流程图、架构图素材清单

## 4. 开发 Agent 配合要求

其他 Agent 在完成工作时，应同步向 Thesis Agent 提供：
- 本轮设计说明
- 关键实现点
- 测试结果
- 失败案例
- 可引用图表素材

## 5. 论文真实性要求

- 论文描述必须与真实实现一致
- 不得虚构未实现模块
- 不得虚构实验结果
- 图表必须能回溯到真实数据和真实界面

## 6. 必备论文文档

- `docs/specs/10-thesis-spec.md`
- `docs/records/THESIS_PROGRESS.md`
- `docs/records/EXPERIMENTS.md`
- `docs/records/FIGURE_ASSETS.md`

## 7. 阶段性输出

### 第一阶段

- 论文大纲
- 系统背景与目标草稿
- 架构初稿

### 第二阶段

- 决策层设计章节草稿
- 数据采集与模型训练材料

### 第三阶段

- 前端实现与工具集成章节草稿
- 界面截图素材

### 第四阶段

- 实验结果分析
- 总结与展望
- 答辩材料支持内容
