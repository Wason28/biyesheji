# 08 里程碑、数据与测试规范

## 1. 里程碑

| 阶段 | 主线任务（决策系统） | 支线任务（数据与模型） | 预计耗时 |
|------|----------------------|------------------------|----------|
| 第一阶段 | 搭建 LangGraph + MCP 框架骨架 | 安装 LeRobot，录制 50-100 组分拣动作数据 | 2-3 周 |
| 第二阶段 | 实现各 Agent 节点与状态流转 | 使用录制数据微调 SmolVLA 模型 | 4-5 周 |
| 第三阶段 | 完成前端界面开发与 MCP 工具集成 | 将 SmolVLA 封装为 MCP 工具，测试推理 | 2-3 周 |
| 第四阶段 | 端到端闭环调试，撰写论文 | 系统稳定性优化，准备答辩演示 | 2-3 周 |

## 2. 数据采集方案

- 场景：桌面物体分拣
- 示例任务：不同颜色方块分类放入指定区域
- 采集方式：`LeRobot control_robot.py` 人工示教
- 数据量：50-100 个成功轨迹
- 数据覆盖：不同物体位置和抓取角度
- 数据格式：`Parquet + 视频帧`

## 3. 测试验证

### 3.1 单元测试

- 感知层各 `MCP` 工具独立调用测试
- 执行层各原子工具独立调用测试
- 配置与错误处理逻辑测试

### 3.2 集成测试

- `LangGraph` 图在模拟工具环境中的逻辑正确性
- 前后端配置联动测试
- 感知到执行的最小闭环流程测试

### 3.3 实物测试

- 至少完成 10 次连续成功的分拣任务
- 记录成功率、失败原因和环境条件

## 4. 阶段交付要求

- 每一阶段结束时必须更新里程碑文档
- 每一阶段必须补充对应测试结果
- 每一阶段必须同步更新论文进度
