# 01 项目基线

## 1. 文档目的

本文档从根需求文档提取项目基线，作为开发执行时的快速对照页，不额外扩展系统边界。

## 2. 项目目标

- 在 PC 端实现模块化具身智能 Agent 原型系统。
- 采用“感知-决策-执行”三层解耦架构。
- 完成用户自然语言指令驱动的机械臂物体分拣闭环任务。

## 3. 核心设计原则

- 模块化解耦：感知、决策、执行独立开发，通过 `MCP` 通信。
- 模型可替换：决策层 `LLM` 与感知层 `VLM` 支持前端动态切换。
- 平台化与可扩展：基于 `MCP` 协议设计工具集，后续可灵活添加新技能。

## 4. 技术基线

| 层级 | 技术方案 | 备注 |
|------|----------|------|
| 决策编排 | `LangGraph (StateGraph)` | 多 Agent 节点闭环流转 |
| 通信协议 | `MCP` | 标准化工具调用与资源访问 |
| 感知 VLM | `MiniMax MCP Vision / OpenAI GPT-4o / Ollama Vision` | 前端可切换 |
| 执行 VLA | `SmolVLA` | 固定，不可前端替换 |
| 机器人框架 | `LeRobot` | 统一硬件控制与数据格式 |
| 前端界面 | `Gradio / Streamlit` | 轻量原型 |
| 硬件平台 | `PC + 单机械臂 + 臂上摄像头` | 本地部署 |
| 目标部署环境 | `Ubuntu` | 当前部署目标 |

## 5. 系统边界

当前版本必须覆盖：
- 用户自然语言输入
- 场景感知与状态读取
- 基于 LangGraph 的多 Agent 决策闭环
- 机械臂动作执行
- 前端模型配置、状态监控和工具展示

当前版本不要求覆盖：
- 多机械臂协同
- 分布式部署
- 执行层基础模型热切换

## 6. 基线约束

- 三层架构不得跨层直接耦合内部实现。
- 决策层、感知层、执行层之间的工具调用必须通过 `MCP` 或显式适配层。
- `SmolVLA` 为当前固定核心执行技能。
- 前端只负责输入、配置和状态展示，不承载核心决策逻辑。
- 后续开发、依赖选择和部署说明优先面向 `Ubuntu`。
