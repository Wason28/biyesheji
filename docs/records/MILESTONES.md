- 当前重点：P0 合同收敛后的接口固化、失败路径集成测试、mock 到真实适配的平滑替换、Ubuntu 运行验证
- 已完成事实：
  - 已建立 `src/embodied_agent` 三层骨架与 `shared/config`
  - 已新增决策、感知、执行三个子系统的 mock 实现
  - 已新增统一启动入口 `src/embodied_agent/app.py`
  - 已完成 P0 第一批、第二批与第三批合同收敛，统一 capability contract 雏形、结构化 `ee_pose`、统一 perception / execution envelope，并补 focused tests 与工厂注入测试
  - 已形成 Ubuntu mock runbook，并完成当前 mock CLI 运行链路验证
- 测试规模已从 10 / 13 / 17 / 21 / 22 / 24 / 37 / 38 / 44 / 54 / 62 逐步扩展到当前全量 67 个通过

  - LeRobot 实装与真实设备联调尚未开始

  - 前端联动、Ubuntu 实机验证与 phase-2 启动前剩余前置项尚未完成

- 支线任务：使用录制数据微调 SmolVLA 模型
- 预计耗时：4-5 周
- 当前状态：主线完成
- 当前基线：统一入口可运行、P0 合同已完成三批收敛、phase-2 决策/感知/执行/前端展示合同能力已落地、当前全量测试 67 个通过
- 当前重点：主线收束完成，外部验证项并行推进
- 主要风险：Ubuntu 实机、前端联动与真实适配策略仍未完成，但不再作为第二阶段主线完成的唯一阻塞条件

## 第三阶段

- 主线任务：完成前端界面开发与 MCP 工具集成
- 支线任务：将 SmolVLA 封装为 MCP 工具，测试推理
- 预计耗时：2-3 周
- 当前状态：已启动，当前处于“前端工程骨架已落地并构建通过，后续进入浏览器联调与验收清单执行”的阶段
- 当前基线：已新增 `backend/contracts.py`、`backend/presenters.py`、`backend/service.py`、`backend/run_registry.py`、`backend/http.py` 与 `adapters/mcp_gateway.py`；已新增 `frontend/` 前端工程，包含 `App.tsx`、5 个工作台面板、`workbench.ts` 状态仓库、`api.ts`/`sse.ts` 接线层与 Vite 代理配置；当前 67 个 pytest 测试通过，且 `frontend/` 已通过 `npm run build`
- 当前重点：先冻结现有 `bootstrap / config / tools / run / runs / events / error` 合同，并围绕已落地前端骨架完成浏览器级联调、截图留痕与最小验收项闭环
- 主要风险：当前 backend 虽已具备最小 HTTP 入口、SSE 回放骨架与终态清理，但仍不是完整 HTTP/SSE/WebSocket 服务；前端虽可构建，但尚无自动化 UI 回归、真实视频流与真实浏览器联调证据
