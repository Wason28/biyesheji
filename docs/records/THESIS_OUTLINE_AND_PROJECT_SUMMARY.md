## 一、论文大纲建议

## 第1章 绪论

### 1.1 研究背景

- 具身智能系统正在从单一算法研究走向感知、决策、执行协同的一体化系统研究。
- 桌面级机械臂场景具有成本低、任务清晰、便于实验复现等特点，适合作为毕业设计落地平台。
- 当前机器人系统常面临模块耦合强、接口不统一、运行过程不可观测、前后端联调困难等问题。

### 1.2 研究意义

- 提出一套适用于桌面级具身智能机器人的“感知—决策—执行”解耦系统方案。
- 通过统一合同、状态流转和前端工作台提升系统可维护性、可扩展性和可验证性。
- 为后续真实模型接入、真实机械臂执行、实验统计和论文答辩演示提供工程基础。

### 1.3 研究目标

- 构建桌面级具身智能机器人一体化原型系统。
- 实现前端交互、感知输入、决策编排、执行反馈与运行态展示的闭环基线。
- 支持真实 LLM/VLM provider 接入，并保留 mock 回退能力。
- 为真实摄像头、真实机器人状态、真实机械臂执行与实体实验预留清晰扩展路径。

### 1.4 论文主要内容与贡献

- 设计了感知层、决策层、执行层三层解耦架构。
- 基于 LangGraph 实现了可追踪的任务状态流转与闭环控制流程。
- 实现了前后端联动的工作台系统，支持配置、运行、事件和快照的统一展示。
- 建立了本地软件级最小闭环验证基线，为后续真实链路接入提供了可复现入口。

## 第2章 相关工作

### 2.1 具身智能系统研究现状

- 具身智能系统通常涉及感知、语言理解、任务规划、动作执行和结果验证等多个环节。
- 多数研究偏向单一模型能力验证，工程化系统集成与可视化运维能力相对不足。

### 2.2 LeRobot 与桌面级机器人生态

- LeRobot 为桌面机械臂数据采集、策略训练和策略评估提供了较完整工具链。
- 在本项目中，LeRobot 对真实数据采集、遥操作、训练与后续真实执行链路具有支撑作用。

### 2.3 LangGraph 与多 Agent/状态流系统

- LangGraph 适合构建有状态、可分阶段执行、支持条件跳转与回退的决策流程。
- 对机器人任务而言，LangGraph 能较好承载任务拆解、执行验证与重试机制。

### 2.4 VLM/LLM 与机器人任务理解

- VLM 可承担场景理解、目标识别、环境描述等感知增强任务。
- LLM 可承担自然语言理解、任务分解、规划建议和策略解释等任务。

### 2.5 SmolVLA 与轻量视觉-语言-动作模型

- VLA 模型适合承接更高层技能执行接口。
- 本项目当前已完成 `run_smolvla` 的实体执行链路接入，可作为执行层高阶技能的真实实现依据。

## 第3章 系统总体架构设计

### 3.1 系统设计目标

- 面向桌面级机器人任务，构建低耦合、可替换、可观测的原型系统。
- 保证前端、后端、模型与执行层之间的边界清晰。

### 3.2 系统总体架构

系统采用三层解耦结构：

- 感知层：负责图像采集、机器人状态获取与场景理解。
- 决策层：负责指令解析、任务分解、状态流转、动作选择与闭环判断。
- 执行层：负责原子动作执行、高层技能调用与结果回传。

同时包含：

- 统一入口与运行时装配层：负责配置加载、模块装配与运行模式切换。
- 前端工作台：负责任务输入、配置管理、运行态展示、日志与事件回放。

### 3.3 分层边界设计

- 前端不承担核心业务决策。
- 决策层不直接访问硬件。
- 感知层只输出标准感知合同，不输出任务规划。
- 执行层只负责动作执行与结果反馈，不承担任务编排。
- 层间通信通过 MCP 或显式适配层完成。

### 3.4 主流程设计

主流程如下：

1. 用户输入自然语言指令。
2. 决策层完成语义解析与任务拆解。
3. 决策层调用感知层获取图像和机器人状态。
4. 决策层根据状态与目标生成动作规划。
5. 执行层执行动作并返回反馈。
6. 决策层对结果进行验证，决定继续、重试或结束。
7. 前端同步展示快照、事件与最终结果。

### 3.5 系统扩展原则

- 优先通过 MCP 工具扩展感知与执行能力。
- 保持 capability contract 稳定。
- 新增能力不能破坏原有主流程和分层边界。

## 第4章 感知层设计与实现

### 4.1 感知层职责

- 获取图像输入。
- 获取机器人状态。
- 对外提供统一结构化输出。

### 4.2 感知合同设计

当前核心能力包括：

- `get_image`
- `get_robot_state`
- `describe_scene`

### 4.3 mock 与真实感知双通道机制

- 未配置真实能力时，系统回退到 mock provider。
- 已预留真实摄像头采集和真实机器人状态读取能力。
- 当前真实输入链路处于逐步接入阶段。

### 4.4 感知层错误处理

- 统一错误类型。
- 支持超时、读取失败、provider 不可用等情况的显式回退。
- 错误结果可回显到前端工作台。

### 4.5 感知层当前状态

- 已具备真实 VLM provider 装配能力。
- 当前仍需进一步完成真实摄像头稳定接入、真实机器人状态长期运行验证与完整实验留痕。

## 第5章 决策层 Agent 设计与实现

### 5.1 决策层总体思路

决策层基于 LangGraph 构建，核心目标是把机器人任务组织为可追踪、可验证、可回退的状态流。

### 5.2 状态流转设计

决策层围绕以下过程组织：

- 交互触发与指令采集
- 语义解析与目标提取
- 多模态环境感知
- 状态置信度评估
- 任务分解与路径规划
- 动作执行控制
- 执行结果验证
- 错误诊断与人工干预
- 任务终态判断与记忆更新

### 5.3 节点能力设计

- 支持任务拆解与阶段推进。
- 支持条件分支、失败回退和补偿控制。
- 支持自然语言对话输出与动作规划并行存在。

### 5.4 模型可替换机制

- 支持 `minimax`、`openai`、`ollama` 等决策 provider。
- 在模型不可用时保留 heuristic fallback。
- 兼顾系统鲁棒性与后续扩展性。

### 5.5 决策层当前状态

- 第二阶段主线已完成。
- 已具备真实 LLM adapter 接入能力。
- 当前已达到“本地软件级决策闭环可验证”状态。

---

## 第6章 执行层设计与实现

### 6.1 执行层职责

- 接收决策层下发的动作请求。
- 执行原子动作或高层技能。
- 返回执行结果与错误信息。

### 6.2 执行能力设计

当前执行层保留以下能力：

- `move_to`
- `move_home`
- `grasp`
- `release`
- `run_smolvla`

### 6.3 执行层安全边界

- 执行层负责动作边界与安全链约束。
- 决策层只做动作选择，不直接操作底层硬件。

### 6.4 当前实现状态

- 当前执行层已完成 SmolVLA 实体执行链路接入。
- 已形成真实机械臂与真实 `SmolVLA` 的协同执行基础。
- 后续仍需继续补充实体任务重复成功率与长时间稳定性验证。

## 第8章 系统验证与结果分析

### 8.1 验证思路

当前验证遵循“先本地最小闭环、再 Ubuntu 工程闭环、再真实感知、真实模型、真实执行、真实实体实验”的推进顺序。

### 8.2 当前已完成验证

- 本地最小前后端闭环验证已完成。
- focused pytest 已通过。
- 前端构建已通过。
- Playwright e2e 已通过。
- 已形成部分浏览器截图与 smoke 结果留档。

### 8.3 当前验证结果

当前已证明：

- 前端、后端、运行时、事件流和快照链路可用。
- 配置保存、工具刷新、run 提交和终态回放可验证。
- 系统具备本地软件级 demo 的可运行、可观测、可回归能力。

### 8.4 当前未完成验证

- Ubuntu 实机完整验收
- 真实视频流长期稳定性
- 真实机器人状态链路长期稳定性
- 复杂任务场景下的重复实体执行统计
- 实体抓取/分拣任务成功率统计

### 8.5 问题与不足

- 当前虽已完成 SmolVLA 实体执行链路接入，但统计性实验与长时间稳定性验证仍不足。
- 长连接服务仍为最小 WSGI + 进程内骨架，不属于生产级服务。
- 实验数据与真实任务统计尚不充分。

## 二、项目总结

## 1. 项目定位

本项目是一个面向桌面级具身智能机器人的感知—决策—执行一体化原型系统。项目以模块化、可扩展、可观测为设计目标，重点解决多模块系统在工程实现中接口不统一、调试困难和运行过程不透明的问题。

## 2. 已完成内容

### 2.1 系统架构方面

- 已完成感知层、决策层、执行层三层骨架建设。
- 已完成统一入口与组合根装配。
- 已完成前后端分离的工作台基线。

### 2.2 决策能力方面

- 已完成第二阶段主线状态流转。
- 已完成 LangGraph 决策流程落地。
- 已支持真实 LLM provider 接入和 heuristic fallback。

### 2.3 感知能力方面

- 已支持真实 VLM provider 装配。
- 已建立图像、机器人状态与场景理解的统一合同。
- 已支持 mock 与真实能力切换。

### 2.4 前端与后端方面

- 已完成最小后端 HTTP/SSE 运行时。
- 已完成前端工作台：配置、工具、运行、事件、快照一体化展示。
- 已支持配置热更新、run 提交、事件订阅、终态回放和错误回显。

### 2.5 验证方面

- 已完成本地最小闭环验证。
- 已完成 focused pytest、前端构建和前端 e2e 基线验证。
- 已形成部分截图、JSON 结果和记录文档。

## 3. 当前完成边界

当前项目的真实完成口径应定义为：

- 第一阶段骨架实现已完成。
- 第二阶段主线收敛已完成。
- 第三阶段前端工作台、后端运行时、真实模型适配与工程化验证基线已完成。

但以下内容不能写成已完成：

- 真实机械臂执行闭环
- 真实实体抓取与分拣实验
- 真实 `SmolVLA` 完整执行验证

## 4. 项目特点

### 4.1 架构清晰

三层解耦设计使系统便于维护、扩展和替换模块。

### 4.2 工程可观测

前端工作台、后端事件流、快照与日志机制提升了系统调试能力。

### 4.3 易于扩展

系统已为真实模型、真实硬件、MCP 工具扩展预留清晰入口。

### 4.4 适合论文落地

系统具备明确的架构设计、实现过程、验证方法和后续实验路线，适合用于毕业设计论文组织。

- 真实实验数据尚少。
- 实体抓取成功率统计尚未形成。

## 三、可直接用于论文的总结表述

本文面向桌面级具身智能机器人场景，设计并实现了一套感知、决策、执行解耦的一体化原型系统。该系统以前端工作台为交互入口，以 LangGraph 为决策编排核心，通过标准化感知合同与执行合同实现多模块协同，并形成了本地软件级最小闭环验证能力。系统当前已完成三层架构骨架、运行时接口、前端工作台、真实 LLM/VLM provider 适配、工程化测试基线以及 SmolVLA 实体执行链路接入。需要说明的是，当前系统虽然已具备真实高阶技能执行事实，但仍未完成大规模实验统计与生产级服务能力建设，因此论文中必须严格区分“实体执行已完成”与“统计验证仍待继续”这两个层面。
